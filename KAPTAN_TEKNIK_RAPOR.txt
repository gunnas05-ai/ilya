2. TEKNOLOJI STACK'I
================================================================================

2.1 BACKEND (API Sunucusu)
--------------------------------------------------------------------------------
  Framework       : NestJS 10.4 (Node.js + TypeScript)
  Calisma Zamani  : Node.js v24+
  Platform        : Express (HTTP) + Socket.IO (WebSocket)
  Veritabani      : PostgreSQL 16+ (birincil) + PostGIS (mekansal sorgular)
  ORM             : TypeORM 0.3.x
  Kimlik Dogrulama: JWT (Access + Refresh Token), TOTP 2FA, API Key

3. VERITABANI GEREKSINIMLERI
================================================================================

3.1 PostgreSQL
--------------------------------------------------------------------------------
  Surum           : PostgreSQL 16+ (zorunlu)
  Veritabani Adi  : kaptan
  Tablo Sayisi    : 55+ tablo (TypeORM autoLoadEntities)
  PostGIS         : Onerilir (mekansal sorgular icin). Yoksa geometry kolonlari
                    double precision'a cevrilmelidir.
  Migration       : TypeORM migration yonetimi mevcut
  Yedekleme       : Gunluk pg_dump + WAL arsivleme onerilir

  Kritik Tablolar:
    - users              (kullanicilar, 16 rol)
    - loads              (yuk ilanlari)
    - bids               (teklifler)
    - tracking_records   (GPS takip kayitlari)
    - escrow_transactions (emanet odemeler)
    - wallets            (dijital cuzdan)
    - invoices           (e-Fatura/e-Arsiv)
    - fuel_stations      (akaryakit istasyonlari)
    - restaurants        (yol ustu lokantalar)
    - marketplace        (arac/ekipman pazaryeri)
    - chat_rooms, chat_messages (mesajlasma)
    - notifications      (bildirimler)
    - system_settings    (sistem ayarlari)

  Tahmini Veri Buyumesi (ilk yil):
    - Kullanici: ~10.000
    - Yuk: ~50.000
    - Takip Kaydi: ~5.000.000
    - Tahmini DB Boyutu: ~5-10 GB (ilk yil)

3.2 Redis
--------------------------------------------------------------------------------
  Surum           : Redis 7.x
  Kullanim        :
    - Bull job queue (e-Fatura, UETDS, toplu islemler)
    - Socket.IO adapter (coklu sunucu WebSocket senkronu)
    - Rate limiting (dagitik)
    - Cache (opsiyonel)

  Tahmini Bellek   : 512 MB - 1 GB

3.3 Kafka (Opsiyonel)
--------------------------------------------------------------------------------
  Surum           : Apache Kafka 3.x
  Kullanim        : Event streaming (yuk olaylari, odeme olaylari, bildirimler)
  Fallback        : EventEmitter2 (in-process) — Kafka yoksa otomatik devreye girer
  Topic'ler       : load.events, bid.events, escrow.events, notification.events,
                    tracking.events, invoice.events

6. API YAPISI
================================================================================

  Protokol        : REST (JSON) + WebSocket (Socket.IO)
  Prefix          : /api/v1/
  Kimlik Dogrulama: Bearer Token (JWT)
  Rate Limit      : 5 istek/dk (login), 100 istek/dk (genel)
  Dokumantasyon   : /api/docs (Swagger UI)

  Temel Endpoint Gruplari:
    /api/v1/auth/*           Kimlik dogrulama (login, register, OTP, MFA, refresh)
    /api/v1/users/*          Kullanici yonetimi
    /api/v1/loads/*          Yuk yonetimi (CRUD, arama, filtre)
    /api/v1/bids/*           Teklif sistemi
    /api/v1/tracking/*       GPS takip (konum kaydi, canli izleme)
    /api/v1/escrow/*         Emanet odeme, anlasmazlik
    /api/v1/wallet/*         Dijital cuzdan, bakiye, islem gecmisi
    /api/v1/finance/*        Gelir/gider, OCR belge tarama
    /api/v1/gib/*            e-Fatura, e-Arsiv, e-Irsaliye
    /api/v1/invoice/*        Fatura yonetimi
    /api/v1/chat/*           Mesajlasma
    /api/v1/notifications/*  Bildirimler
    /api/v1/fuel-stations/*  Akaryakit istasyonlari
    /api/v1/restaurants/*    Restoranlar
    /api/v1/marketplace/*    Arac pazaryeri
    /api/v1/part-market/*    Yedek parca pazaryeri
    /api/v1/vehicles/*       Arac filosu yonetimi
    /api/v1/routing/*        Rota planlama
    /api/v1/ai-matching/*    AI yuk esleme
    /api/v1/integrations/*   API Key + Webhook yonetimi
    /api/v1/admin/settings/* Sistem ayarlari (KVKK metinleri dahil)
    /api/v1/health           Saglik kontrolu (Kubernetes liveness/readiness)

  WebSocket Olaylari:
    load.created, load.updated, load.cancelled
    bid.placed, bid.accepted, bid.rejected
    tracking.update, tracking.status_change
    escrow.created, escrow.released, escrow.dispute
    invoice.created, invoice.status_change
    notification.new, notification.read
    wallet.balance_change, wallet.transaction
    chat.message, chat.typing

7. DEPLOYMENT (DOCKER)
================================================================================

7.1 Backend Dockerfile
--------------------------------------------------------------------------------
  Base Image      : node:20-alpine
  Build           : Multi-stage (tsc derlemesi → dist/main.js)
  Production Run  : node dist/main.js
  Port            : 3000 (EXPOSE)
  Saglik Kontrolu : /api/v1/health

7.2 Web Dockerfile
--------------------------------------------------------------------------------
  Base Image      : node:20-alpine
  Build           : next build (static + dynamic export)
  Production Run  : next start -p 3000
  veya            : next export (tamamamen statik, Nginx'ten servis)
  Port            : 3000 (varsayilan)

7.3 docker-compose.yml (ornek)
--------------------------------------------------------------------------------
  services:
    backend:
      build: ./backend
      ports: ["3000:3000"]
      depends_on: [postgres, redis]
      environment:
        - DB_HOST=postgres
        - REDIS_HOST=redis

    web:
      build: ./web
      ports: ["80:3000"]
      depends_on: [backend]

    postgres:
      image: postgis/postgis:16-3.4
      volumes: [pgdata:/var/lib/postgresql/data]

    redis:
      image: redis:7-alpine
