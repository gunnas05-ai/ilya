================================================================================

2.1 [BACKEND] Refresh Token Rotasyonu Yok
    Dosya: backend/src/auth/auth.service.ts:153-166
    Sorun: Refresh token kullanıldıktan sonra rotate edilmiyor. Token çalınırsa

    Çözüm: Her refresh'te yeni token üretip eskisini invalidate edin.

2.2 [BACKEND] SQLite'da synchronize:true
    Dosya: backend/src/database-config.ts:35
    Sorun: PostgreSQL yoksa SQLite'a düşüyor ve synchronize:true ile tüm
           şema değişikliklerini otomatik uyguluyor. Kolon/tablo SİLEBİLİR.
    Çözüm: SQLite'ta da migration kullanın veya synchronize'ı kaldırın.

2.3 [BACKEND] Eksik Index'ler
    Dosyalar: loads/load.entity.ts, bids/bid.entity.ts, escrow/escrow.entity.ts
    Sorun: fromCity, toCity, loadType, vehicleType, creatorId, loadId, carrierId
           — hiçbirinde @Index() yok. Büyük veride full table scan.
    Çözüm: Sorgulanan tüm kolonlara @Index() ekleyin.

2.4 [BACKEND] Input Validation Eksikliği
    Dosyalar: Çoğu controller
    Sorun: @Body() body: any kullanılıyor. class-validator DTO'ları sadece
           AuthController ve UsersController'da var. Hatalı veri runtime'da patlıyor.
    Çözüm: Tüm endpoint'lere typed DTO + class-validator ekleyin.

2.5 [BACKEND] CommonModule @Global() Aşırı Kullanımı
    Dosya: backend/src/common/common.module.ts:39
    Sorun: 16 servis + Kafka bağlantısı global yapılmış. Bağımlılık grafiği
           görünmez hale gelmiş.
    Çözüm: @Global()'i kaldırın, modülleri explicit import edin.

2.6 [MOBİL] Tüm Ekranlar Eager Loading
    Dosya: mobile/src/navigation/index.tsx:15-72
    Sorun: 62 ekranın tamamı statik import ile başlangıçta yükleniyor.
           Başlangıç süresi çok uzun.
    Çözüm: React.lazy() + Suspense ile lazy loading uygulayın.

2.7 [MOBİL] 30+ Ekran Servis Katmanını Atlıyor
    Dosyalar: HomeScreen, AdminPanelScreen, WalletScreen, TaxDashboardScreen vb.
    Sorun: apiClient doğrudan ekranlardan çağrılıyor. Response unwrap mantığı
           her ekranda tekrar ediliyor. Test edilemez.
    Çözüm: Tüm API çağrılarını servis katmanına taşıyın.

2.8 [MOBİL] Zustand Store'lar React Query Yerine Server State Tutuyor
    Dosyalar: loadAcceptStore, trackingStore, analyticsStore, returnLoadStore,
             poiStore, carrierStore
    Sorun: React Query kurulu ama kullanılmıyor. Her store kendi loading/error
           state'ini yönetiyor. Cache yok, refetch yok.
    Çözüm: Server state'i React Query'ye taşıyın, Zustand'ı sadece UI state
           için kullanın.

2.9 [MOBİL] Zero React.memo Kullanımı
    Genel: Tüm kod tabanı
    Sorun: Hiçbir bileşende React.memo yok. Her parent render'da tüm child'lar

2.10 [MOBİL] Inline Style'lar Her Render'da Yeniden Oluşuyor

    Dosya: backend/.env
    Risk: JWT_SECRET, JWT_REFRESH_SECRET, ENCRYPTION_KEY, IYZICO anahtarları

    Dosya: mobile/src/services/securityService.ts:43-55
    Risk: isSecureConnection sadece https:// prefix'i kontrol ediyor. Zararlı
          CA sertifikasıyla MITM yapılabilir.
    Çözüm: react-native-ssl-pinning veya TrustKit entegre edin.

4.3 [MOBİL] Root/Jailbreak Tespiti Boş
    Dosya: mobile/src/services/securityService.ts:17-24
    Risk: checkDeviceIntegrity() içi boş — her zaman true dönüyor.
    Çözüm: Gerçek root/jailbreak tespiti ekleyin (SafetyNet, JailMonkey).

4.4 [BACKEND] EscrowAdminController Yetkisiz
    Dosya: backend/src/vehicles/vehicles.controller.ts:99-109
    Risk: admin/escrow endpoint'leri sadece JWT kontrolü yapıyor, rol kontrolü
          yok. Herhangi bir authenticated kullanıcı escrow'u toggle'layabilir.
    Çözüm: @Roles('super_admin', 'admin') ekleyin.

4.5 [MOBİL] Crash Report Spoofing
    Dosya: mobile/src/services/crashReporting.ts, backend/src/common/crash-report.controller.ts:18
    Risk: Crash report endpoint'i auth gerektirmiyor. userId ve userEmail client
          tarafından gönderiliyor — herkes istediği kullanıcı adına crash report
          gönderebilir.
    Çözüm: userId'yi JWT'den alın, client'tan kabul etmeyin.

4.6 [BACKEND] SozlesmeController Korumasız
    Dosya: backend/src/common/sozlesme.controller.ts:27-29
    Risk: Admin endpoint'leri inline rol kontrolü yapıyor ama req.user undefined

          Hata mesajı stack trace içeriyor olabilir.
    Çözüm: @UseGuards(AuthGuard('jwt'), RolesGuard) + @Roles('super_admin')
           ekleyin.
